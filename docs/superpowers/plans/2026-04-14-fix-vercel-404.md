# Fix Vercel React Router 404 Issue Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix the `404: NOT_FOUND` error on Vercel when directly accessing specific routes (like `/kingdomofkumar`).

**Architecture:** 
1. React uses client-side routing. When you navigate to `https://www.hireschema.com/kingdomofkumar` directly, Vercel looks for a physical file named `kingdomofkumar.html` on the server.
2. Since it doesn't exist (it's an SPA), Vercel returns a 404.
3. We need to add a `rewrites` rule in `vercel.json` to tell Vercel to redirect all non-API traffic to `index.html` so React Router can handle it.

**Tech Stack:** Vercel config.

### Task 1: Update vercel.json

**Files:**
- Modify: `vercel.json`

- [ ] **Step 1: Add the rewrites configuration**
```json
{
  "installCommand": "npm install",
  "buildCommand": "npm run build",
  "outputDirectory": "dist",
  "framework": "vite",
  "crons": [
    {
      "path": "/api/cron/daily-alerts",
      "schedule": "30 2 * * *"
    }
  ],
  "rewrites": [
    {
      "source": "/(.*)",
      "destination": "/index.html"
    }
  ]
}
```

Wait, `rewrites` must not catch `/api/*` requests, otherwise the backend functions will break!

```json
{
  "installCommand": "npm install",
  "buildCommand": "npm run build",
  "outputDirectory": "dist",
  "framework": "vite",
  "crons": [
    {
      "path": "/api/cron/daily-alerts",
      "schedule": "30 2 * * *"
    }
  ],
  "rewrites": [
    {
      "source": "/api/(.*)",
      "destination": "/api/$1"
    },
    {
      "source": "/(.*)",
      "destination": "/index.html"
    }
  ]
}
```
